1) การเชื่อมต่อ Keycloak (Local และ VM)

ได้ตั้งค่า Keycloak ให้สามารถใช้งานได้ทั้งบนเครื่อง Local และบน VM พร้อมปรับ redirect URL, token และโครงสร้างการ init ให้ทำงานถูกต้อง
สิ่งที่ได้เรียนรู้: เข้าใจการทำงานของ token, session storage และวิธีป้องกันการ init ซ้ำมากขึ้น

2) การ Fetch Declaration Status

จากเดิมระบบดึงข้อมูลเพียง endpoint เดียว จึงปรับให้ดึงสอง endpoint (study-plans และ declared-plan) เพื่อให้ข้อมูลครบและตรงตาม requirement
สิ่งที่ได้เรียนรู้: การทำงานของ asynchronous fetch หลายตัว การจัดการ DOM ไม่ให้โหลดซ้ำ และลด request ที่ไม่จำเป็น

3) Reservation Period

ได้พัฒนา logic ตรวจสอบว่าอยู่ในช่วง Reservation Period หรือไม่ เพื่อให้ระบบแสดงผลถูกต้อง เช่น เปิด–ปิดปุ่มจอง และแจ้งเตือนเมื่ออยู่นอกช่วงเวลา
สิ่งที่ได้เรียนรู้: การจัดการเวลาในระบบจริง การแยก time function ออกจาก container และแก้ปัญหา edge case เรื่องเวลา

4) Course Offerings และ Course Reservations

พัฒนาส่วนแสดงรายการรายวิชาและข้อมูลการจอง โดยปรับ layout ให้ถูกต้อง รองรับกรณีข้อมูลว่าง (empty state) และเพิ่มสถานะโหลดข้อมูล
สิ่งที่ได้เรียนรู้: การออกแบบ DOM หลายส่วนพร้อมกัน การแยกโครงสร้าง render และการจัดการสถานะของ UI หลายแบบ

5) การทดสอบระบบด้วยการตั้งค่าเวลาใน DevTools

ได้ใช้เทคนิคปรับเวลาใน DevTools เพื่อทดสอบการทำงานของระบบช่วง Reservation Period เช่น ทดสอบเมื่ออยู่ในช่วง–นอกช่วงจอง โดยไม่ต้องรอเวลาจริง
สิ่งที่ได้เรียนรู้: เข้าใจผลกระทบของเวลาในฝั่ง Frontend การ Mock เวลาเพื่อทดสอบระบบ และวิธีตรวจสอบความถูกต้องของ logic ที่อิงตามเวลา
