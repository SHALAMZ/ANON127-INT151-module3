สิ่งที่ได้จากโค้ด Date/Time

เข้าใจความต่างระหว่าง
new Date() → เวลา local ของเครื่อง
Date.now() → ได้ค่า timestamp (ms) แบบตัวเลข

รู้ว่า "2025-11-18T10:30:00Z" (มี Z) เป็นเวลามาตรฐาน UTC
ส่วน "2025-11-18T10:30:00" (ไม่มี Z) จะถูกตีความเป็นเวลาตาม timezone เครื่อง

การสร้าง Date แบบตัวเลข new Date(2025, 12, 30, 15, 25)
→ เดือนใน JS เริ่มที่ 0, เดือน 12 = มกราคมปีถัดไป
(อันนี้ทำให้รู้ว่า Date ใน JS เดือนจะ +1 เสมอ)

การเปรียบเทียบวันที่ด้วย === จะ ไม่เท่ากัน
เพราะ Date เป็น object คนละตัว
แต่ถ้าใช้ .getTime() จะเทียบ timestamp ได้ถูกต้อง

date1 > date2 และ date1 < date2 ใช้ได้
เพราะ JS จะเปรียบเทียบโดยแปลงเป็น timestamp อัตโนมัติ

ใช้ .getTime() เพื่อใช้ทำ logic ช่วงเวลา เช่น

เช็กว่าอยู่ในช่วง booking หรือไม่

เช็ก start/stop time ได้แม่นกว่าเช็ก object
